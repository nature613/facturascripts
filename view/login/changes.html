Últimos cambios de la versión {$fsc->version()}:

  • 02/07/2013:
    • Ya se pueden hacer regularizaciones trimestrales de IVA.
    • Desde Contabilidad > caja ahora se muestra la diferencia de dinero (las ganancias) incluso de las cajas no cerradas.
    • Solucionado el bug que hacía que no se pudiese "instalar" FacturaScripts cuando ya existía la tabla agentes.
  • 18/06/2013:
    • Si el usuario no tiene marcada una página por defecto, se selecciona la primera página importante de su lista.
    • Nuevas páginas de acceso denegado, página no encontrada y error en la base de datos.
    • Cuando un usuario accede a una página a la que no tiene acceso se producía un bug que hacía que la página no terminase de cargar. El bug estaba en las funciones url() de los controladores. Para solucionarlo he añadido una comprobación más antes de proporcionar la url.
    • Añadida encuenta on-line.
    • La página de cada artículo ya permite navegar por todos los albaranes y facturas donde figura ese artículo.
  • 14/06/2013:
    • Ahora se puede cambiar la serie de albaranes no facturados.
    • Los asientos de ejercicios ya cerrados se bloquean automáticamente (se marcan como no editable).
    • A los artículos bloqueados se les elimina el stock automáticamente.
    • TPV genérico permite configurar si quieres que se impriman las referencias o las descripciones en los tickets.
    • Las facturas de clientes incluyen la dirección y teléfonos de los mismos.
    • Mejoradas las hojas de estilo.
  • 22/05/2013:
    • Añadido soporte experimental para WooCommerce.
    • Añadido control de duplicados. Ahora cuando se envían dos peticiones de un mismo formulario (de nuevo asiento, nuevo albarán o lo que sea) ya sea por fallo del navegador, por mala leche del usuario o por hacer doble clic sobre el botón guardar, se guarda el primero, pero la segunda petición y siguientes no producen elementos duplicados.
    • Los asientos, albaranes y facturas ahora muestran un mensaje de error si encuentran duplicados.
    • El libro mayor de cada subcuenta ahora incluye las sumas del debe y del haber, para facilitar cálculos.
    • El TPV de restaurante permite ahora imprimir un ticket por cada impresora.
    • Pequeñas modificaciones para dar más información al usuario y ofrecer una experiencia más sencilla.
  • 29/04/2013:
    • Añadido soporte para MySQL.

  • 22/04/2013:
    • Ya se pueden modificar los asientos.
    • Pequeñas modificaciones en el HTML.
  • 15/04/2013:
    • Ya se pueden imprimir albaranes de cliente, en formato a5. Aunque en realidad es un a4 hasta la mitad del folio, para engañar al navegador y que lo imprima bien.
    • Nueva clase fs_pdf para genear PDFs. Proporciona diversas función extra como guardar PDFs, centrar texto o crear tablas de forma más sencilla.
    • El formulario de factura de cliente ya permite modificar el número.
